    The profinite Demushkin group D0 (Stage C, work order C4) #

    This file realises the manuscript's standard rank-three dyadic Demushkin group D₀ = ⟨a, s, y ∣ a² s⁴ [s,y] = 1⟩_{pro-2} (reference/q2_manuscript.tex, l.208, Proposition prop:markedDem; equivalently the HNN form Y⁻¹SY = T = S⁻³A⁻², l.814, eq:D0HNN) as the cofiltered inverse limit, in ProfiniteGrp, of all finite marked Demushkin 2-group quotients — exactly mirroring the Stage-C construction of Ttame in Boundary/Ttame.lean, but with the 3-generator Demushkin marking (a, s, y) and the defining relation a² s⁴ [s,y] = 1 in place of the 2-generator tame marking (s, t) with τ^σ = τ².

    The finite index objects D0MarkedFinite, their morphisms D0Hom, the category, D0Hom_ext, and D0Iso are reused from Boundary/Nielsen.lean (work order C4, the Nielsen-transform file). This file adds the limit-theoretic content:

    • skeletalization of an arbitrary Demushkin-marked finite 2-group (d0Skeletalize, d0SkelEquiv, d0SkeletalizeHomOut, d0SkelHomBetween);
    • the trivial object and binary subdirect objects, giving IsCofiltered D0MarkedFinite;
    • d0Diagram : D0MarkedFinite ⥤ ProfiniteGrp, D0 := Limits.limit d0Diagram, the bridge d0Iso to mathlib's explicit subtype model and D0_ext;
    • the universal generators d0A, d0S, d0Y : D0, their projection-eval lemmas, the coordinatewise Demushkin relation D0_rel, and dense generation d0Gen_topologically_generate.

    Deferred: D0 ≅ PiGrp #

    The profinite Nielsen comparison D0 ≅ PiGrp (manuscript Cor. 3.12, cor:relativeDemushkin) is deferred to a follow-up file: it will follow from Nielsen's categorical equivalence D0_Pi_index_equiv : D0MarkedFinite ≌ PiMarkedFinite together with limit functoriality, once the parallel Boundary/PiGroup.lean (PiGrp := Limits.limit piDiagram) is available. We do not import PiGroup.lean here.

    Instance gotcha (inherited from Nielsen.lean / AdmissibleIndex.lean) #

    Fin n carries the global Fin.instMul, not the transported group structure D.groupFin. We thread D.groupFin explicitly (@, letI) through every Fin-level construction so Fin.instMul is never synthesized (mirroring tameSkelHomBetween).
