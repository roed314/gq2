    Candidate exact-image concrete data closers #

    This owned sidecar refines the aggregate residual centralCover_candidateConcreteExactImageData from CandidateExactImageConcreteClosers.lean.

    The paper-side content is split along Section 8 and the Pro validation note:

    • quotient-frame compatibility for postcomposition (lem:covertransform);
    • semantic R-lift obstruction data on the exact-image middle index (def:Xgamma, lem:obstructionseparation);
    • Frattini-cover exact-image data for the canonical affine fibres (prop:finalfourier).

    No theorem below uses the aggregate axiom centralCover_candidateConcreteExactImageData.

    Boundary-frame compatibility for the quotient map Y -> B = Y/R.

    For an arbitrary bare T : CentralCoverElementaryQuotientTowerData B, the top and middle targets do not expose definitionally identical boundary codomains. This type-safe formulation records exactly what Section 8 needs: any top boundary equation is preserved after quotient postcomposition to the middle frame.
