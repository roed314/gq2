    Candidate exact-image Route67 closers #

    Route67 is candidate-side only. It gives the active integrated route a provider with the same public call shape as Route63,

    B T Q -> Nonempty (CentralCoverCandidateLiftPartitionData B)

    but it does not route through the two broad exact-image packet axioms currently visible in Route66:

    • centralCover_candidateObstructionMapData;
    • centralCover_candidateFrattiniExactImageCardinalityData.

    Instead it reuses the existing Route39 manuscript-shaped exact-image split:

    • one selected scalar-pushout family, additive in lambda : X_R, which builds the actual middle R-valued obstruction map (lem:obstructionseparation);
    • the final Frattini/Fourier zero-locus count for that selected obstruction (prop:finalfourier), plus the theorem-level regular affine-fibre torsor count.

    Thus the Route67 provider strictly lowers the two Route66 candidate atoms to the selected scalar-obstruction and selected Frattini/Fourier packets already present in the API. No theorem in this file calls the broad packet axioms.
