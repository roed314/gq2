    Candidate Route197 pointwise lowering of the Route196 candidate leaves #

    Route196 uses CandidateRoute194ActualLowerClosers, whose candidate axiom closure is the two current Section 8 semantic packets

    • centralCover_candidateMiddleScalarObstructionEvaluatorData;
    • centralCover_candidateFrattiniAffineFiberCoordinateData.

    This sidecar keeps the exact Route196 downstream B T Q provider shape, but routes the candidate lift partition through the existing Route190/Route146 pointwise scalar--Frattini coordinate layer. Consequently an integrated route using the provider below should remove both Route196 candidate leaves from the candidate branch and replace them by the pointwise Route146 residual surface:

    • pointwise nonzero scalar-pushout values;
    • pointwise nonzero-distinct scalar additivity;
    • pointwise Frattini base coordinates;
    • pointwise Frattini fibre coordinates;
    • pointwise Frattini fibre realization;
    • pointwise realization-after-coordinate;
    • pointwise coordinate-after-realization.

    No target, kernel, pushout, local/CFT, or terminal data is introduced here.
