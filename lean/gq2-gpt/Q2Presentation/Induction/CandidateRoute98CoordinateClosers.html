    Candidate Route98 coordinate closers #

    This sidecar is candidate Section 8 only. It does not edit any integrated route file.

    Route98 currently consumes

    B T Q -> Nonempty (CentralCoverCandidateLiftPartitionData B)

    through centralCover_candidateLiftPartitionData_from_route94. The candidate closure of that provider still exposes the two current Section 8 packets:

    • centralCover_candidateMiddleScalarObstructionEvaluatorData;
    • centralCover_candidateFrattiniAffineFiberCoordinateData.

    This file lowers those packets to the coherent route-needed data isolated in CandidateScalarFrattiniCurrentProofs: one selected scalar-pushout evaluator with linearity for that same choice, and one selected Frattini coordinate/realization/inverse tuple attached to the induced middle obstruction.
