    Current candidate scalar/Frattini proof refinement #

    This sidecar refines the five route8 candidate scalar/Frattini residuals without editing the routed files.

    The current scalar API asks for a value packet and then asks for linearity for an arbitrary chosen value packet. The manuscript construction only needs one coherent scalar-pushout evaluator. The current Frattini API similarly asks for coordinate maps, realization maps, and inverse laws separately; the route only needs one coherent coordinate/realization equivalence.

    The replacement residuals below record exactly those coherent choices and then reassemble the same route-ready lift-partition theorem shape consumed by the integrated routes.
