    Candidate scalar/Frattini route15 proofs #

    This sidecar gives a route14-shaped candidate lift-partition provider without using the eight pointwise scalar/Frattini packet axioms exposed by route14:

    • centralCover_candidateMiddleScalarPushoutValueAtData;
    • centralCover_candidateMiddleScalarPushoutAdditivityData;
    • centralCover_candidateMiddleScalarPushoutHomogeneityData;
    • centralCover_candidateFrattiniBaseCoordinateData;
    • centralCover_candidateFrattiniFiberCoordinateData;
    • centralCover_candidateFrattiniFiberRealizationData;
    • centralCover_candidateFrattiniRealizeLiftCoordinateData;
    • centralCover_candidateFrattiniLiftCoordinateRealizeData.

    The available API still does not construct the scalar evaluator or Frattini coordinate maps from first principles. The theorem below therefore routes through the two smaller current semantic inputs:

    • centralCover_candidateMiddleScalarObstructionEvaluatorData;
    • centralCover_candidateFrattiniAffineFiberCoordinateData.

    No routed file is edited here.
