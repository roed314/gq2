    Candidate scalar/Frattini route38 proofs #

    Route38 keeps the Route37 drop-in provider shape

    B T Q -> Nonempty (CentralCoverCandidateLiftPartitionData B)

    but it does not call the two Route37 candidate residual axioms

    • centralCover_candidateScalarFrattiniRoute37ScalarData;
    • centralCover_candidateRoute37FrattiniExactImagePartitionData.

    Instead it exposes smaller Section 8 packets.

    • Scalar side: pointwise scalar-pushout tests indexed by lambda : B.XR, together with additivity for the selected pointwise family. This is the scalar-pushout separation step of lem:obstructionseparation.
    • Frattini side: the exact-image central-cover transform is kept only in its finite cardinal form. The existing finite API then recovers the partition by centralCover_candidateFrattiniExactImageSurjectivityData_of_cardinality.

    This is not a full closure of the candidate calculation: the actual word-level scalar pushout values and the finite central-cover transform remain manuscript inputs. The lowering is strict at the Route37 frontier because the exported provider's axiom closure no longer contains either Route37 residual name.
