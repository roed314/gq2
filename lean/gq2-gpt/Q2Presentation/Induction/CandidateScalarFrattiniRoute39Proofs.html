    Candidate scalar/Frattini route39 proofs #

    Route39 keeps the Route38 integration shape

    B T Q -> Nonempty (CentralCoverCandidateLiftPartitionData B)

    but avoids the three Route38 candidate residual names.

    • Scalar side: instead of pointwise values plus additivity for an arbitrary externally supplied family, Route39 asks for one selected scalar-pushout value family together with additivity for that same family. Homogeneity over ZMod 2 is theorem-level from the Route36 argument.
    • Frattini side: instead of the finite exact-image transform packet, Route39 asks for the final Fourier zero-locus count. The sigma-cardinality of the regular affine R-fibres is theorem-level, so this recovers the Route38 cardinal transform without using the Route38 transform axiom.

    This is a strict lowering of the Route38 candidate frontier: the exported provider's axiom closure names the two Route39 packets below, not centralCover_candidateRoute38ScalarPushoutValueAtData, centralCover_candidateRoute38ScalarPushoutAdditivityData, or centralCover_candidateRoute38FrattiniExactImageTransformData.
