    Candidate scalar/Frattini Route85 closers #

    This sidecar is a drop-in candidate-side refinement for the Route84 work route. Route84 consumes CandidateScalarFrattiniRoute81Closers, whose axiom closure exposes the five semantic scalar/Frattini packets

    • centralCover_candidateMiddleScalarPushoutValueData;
    • centralCover_candidateMiddleScalarPushoutLinearityData;
    • centralCover_candidateFrattiniLiftCoordinateData;
    • centralCover_candidateFrattiniRealizationData;
    • centralCover_candidateFrattiniCoordinateInverseData.

    Route85 does not use those five packets. Instead it routes through the finer manuscript-shaped API isolated in CandidateScalarFrattiniActualClosers: pointwise scalar-pushout tests, additivity and homogeneity for the selected scalar family, a base Frattini coordinate, a fibre coordinate over that base, a curried fibre realization, and the two inverse laws.

    The public theorem

    centralCover_candidateLiftPartitionData_from_route85

    has the same B T Q call shape as the Route81 provider, so an integrated work route can switch to it without changing downstream Fourier/recursion code.
