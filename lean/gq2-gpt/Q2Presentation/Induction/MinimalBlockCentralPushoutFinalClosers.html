    The existing group-tower/provenance APIs expose the tower R, M, T, C and the actions, but they do not expose an actual quotient object K_lambda = K / ker lambda or its square/commutator maps. The residual below is therefore the missing central-extension pushout construction itself. It is strictly stronger than the current semantic packet: it records one same-family phi_lambda, the actual square map q_lambda, its diagonal and polar identification with phi_lambda, naturality, and the simple-head determinant annihilation/radical conclusions.

    The current semantic packet is then just the projection forgetting everything except phi.

    Type

    Actual scalar central-extension pushout data over a pre-head Section 7 tower.

    For every invariant nonzero lambda : R -> F_2, the manuscript forms the central pushout K_lambda = K / ker lambda. The current Lean tower does not carry K_lambda as a quotient group object, so this packet records the exact semantic data used downstream:

    • phi is the self-duality/polar pairing induced by the commutator pairing;
    • square is the square map q_lambda;
    • square_diagonal and polar_symm identify that square and polar with the same phi_lambda;
    • phi_natural, square_annihilates_on_T, and square_radical_subset_T are the same-family prop:simpleheaddet conclusions.

    This is not a zero-map shortcut: the square map is an explicit field tied to phi by diagonal and polar identities, and the extraspecial comparison below is proved by quadratic-form extensionality from those identities.

    • phi : Module.Dual (ZMod 2) D.R → D.M →ₗ[ZMod 2] Module.Dual (ZMod 2) D.M

      The self-duality pairing induced by the scalar central pushouts.

    • square : Module.Dual (ZMod 2) D.R → Quadratic.QuadF2 D.M

      The actual square map of the same scalar central pushout family.

    • square_diagonal (lam : Module.Dual (ZMod 2) D.R) (m : D.M) : (self.square lam).form m = ((self.phi lam) m) m

      The actual square map evaluates on the diagonal of phi_lambda.

    • polar_symm (lam : Module.Dual (ZMod 2) D.R) (m m' : D.M) : ((self.square lam).polar m) m' = ((self.phi lam) m) m' + ((self.phi lam) m') m

      The actual polar map is the symmetrized phi_lambda pairing.

    • phi_natural (lam : Module.Dual (ZMod 2) D.R) : lam ≠ 0 → (∀ (c : D.C), lam ∘ₗ ↑(D.actR c) = lam) → ∀ (c : D.C) (m m' : D.M), ((self.phi lam) ((D.actM c) m)) ((D.actM c) m') = ((self.phi lam) m) m'

      Naturality/equivariance of the same central-pushout pairing.

    • square_annihilates_on_T (lam : Module.Dual (ZMod 2) D.R) : lam ≠ 0 → (∀ (c : D.C), lam ∘ₗ ↑(D.actR c) = lam) → Annihilates (self.square lam) D.T

      q_lambda|_T = 0 and b_lambda(T,M)=0 for the actual square map.

    • square_radical_subset_T (lam : Module.Dual (ZMod 2) D.R) : lam ≠ 0 → (∀ (c : D.C), lam ∘ₗ ↑(D.actR c) = lam) → ∀ (m : D.M), (∀ (m' : D.M), ((self.square lam).polar m) m' = 0) → m ∈ D.T

      The polar radical of the actual square map is contained in T.
