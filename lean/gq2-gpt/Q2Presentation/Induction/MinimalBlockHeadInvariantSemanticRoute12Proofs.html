    Route12 head/invariant semantic refinements for Section 7 #

    This sidecar keeps the integrated route files untouched. It reuses the theorem-level simple-head/scalar closers to replace the four route12 head/invariant semantic packets

    • minimalBlock_rawHeadDualInvariantZeroData;
    • minimalBlock_rawHeadNonzeroStableEqTopData;
    • minimalBlock_rawInvariantCharacterData;
    • minimalBlock_rawInvariantFunctionalsKillTData.

    The replacement is the manuscript split from Section 7: invariant hyperplanes contain T, the quotient head is irreducible and nontrivial, and the lower Frattini layer carries a nonzero invariant character. The exported group-tower provider below is compatible with IntegratedCurrentRouteProofs12.

    One same-raw-tower semantic packet for the Section 7 simple-head/invariant layer.

    The fields are the manuscript-shaped inputs behind lem:simplehead and the lower scalar layer:

    • invariant hyperplanes in M contain T;
    • every nonzero stable submodule of the quotient head is the whole head;
    • the quotient-head action is not pointwise trivial;
    • R has a nonzero C-invariant character.

    This is intentionally narrower than the old route-facing packet names and is attached to one selected raw tower.

    • no_invariant_hyperplane_misses_T : ¬∃ (phi : Module.Dual (ZMod 2) D.M), phi ≠ 0 ∧ (∀ (c : D.C), phi ∘ₗ ↑(D.actM c) = phi) ∧ ∃ t ∈ D.T, phi t ≠ 0
    • eq_top_of_nonzero_stable (U : Submodule (ZMod 2) (D.M ⧸ D.T)) : minimalBlock_rawHeadStable D U → (∃ x ∈ U, x ≠ 0) → U = ⊤
    • not_pointwise_trivial : ¬∀ (c : D.C) (x : D.M ⧸ D.T), (minimalBlock_rawHeadAction D c) x = x
    • exists_nonzero_invariant : ∃ (character : Module.Dual (ZMod 2) D.R), character ≠ 0 ∧ ∀ (c : D.C), character ∘ₗ ↑(D.actR c) = character
