    Route34 minimal-block kernel/head refinement #

    Route33 currently uses the route31 kernel/head provider. Route31 keeps the concrete K, Phi(K), R, M, T, C tower split into the fine Section 7 leaves, but it still obtains the final simple-head/lower-character package from the monolithic same-raw-tower residual minimalBlock_invariantActualProofs2Data.

    This sidecar exports a route31-compatible provider whose data shape is identical to route31, while sourcing that invariant package through the route29 route21-style simple-head split:

    • invariant hyperplanes contain T;
    • nonzero stable quotient-head submodules are top;
    • the quotient-head action is not pointwise trivial;
    • the lower Frattini layer has a nonzero invariant character.

    No routed file is edited. This is a genuine refinement of the route33 kernel/head frontier: the route33-listed monolithic invariant residual is no longer in the closure of the exported provider.
