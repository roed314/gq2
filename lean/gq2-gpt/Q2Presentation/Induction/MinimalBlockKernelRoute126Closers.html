    Route126 kernel closers #

    This sidecar leaves the promoted Route126 files untouched. It supplies a drop-in kernel provider with the same downstream consumer shape as minimalBlock_groupTowerData_from_kernelRoute124Closers:

    Nonempty (MinimalBlockGroupTowerData p chief)

    Route124/Route126 public currently use two Section 7 kernel leaves:

    • minimalBlock_lowerGroupModuleData;
    • minimalBlock_invariantActualProofs2Data.

    The theorem below replaces those two leaves by the same-tower provenance packet

    • minimalBlock_provenanceSemanticData.

    This packet is manuscript-shaped: it fixes the concrete K, R = Phi(K), M = K/R, T, C provenance and carries the head and lower-character data on the raw tower extracted from that same provenance.
