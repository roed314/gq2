    Route50 minimal-block kernel/module invariant provenance #

    This sidecar is route-neutral: it does not edit the routed capstone or any existing integrated route file.

    Route49 still gets its Section 7 kernel provider from Route48, whose active kernel leaf is the broad packet

    • minimalBlock_groupInvariantRouteData.

    The current nonterminal branch only consumes the resulting MinimalBlockGroupTowerData. For that purpose, the needed Section 7 kernel data are:

    • one concrete provenance tower for R = Phi(K) <= K, M = K/R, T, C, actions, bridge, and nonzero head;
    • direct dual-invariant vanishing (M^vee)^C = 0 on that tower;
    • one raw nonzero invariant character on the same Frattini layer.

    The packet below deliberately does not store quotient-head simplicity or the stronger lower scalar quotient packet; those are not consumed by the Route49 kernel branch.
