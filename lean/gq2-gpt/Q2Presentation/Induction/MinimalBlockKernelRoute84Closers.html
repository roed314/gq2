    Route84 kernel closers for the minimal-block route #

    This sidecar advances the Route86 kernel frontier without editing the integrated Route86 file. Route83 already projects the simple-head fields from the coherent actual invariant packet minimalBlock_invariantActualProofs2Data; the remaining raw tower part still used seven separate Route43 literal/object/action leaves.

    Route84 replaces those seven leaves by one same-tower literal packet containing the Section 7 objects

    • R = Phi(K),
    • M = K/Phi(K),
    • T = (K cap S)/Phi(K),
    • C = Y/K,

    together with the transported actions and T-stability on those same objects. The downstream Route55 consumer still sees the same raw object/action/bridge shape, so the public provider is a drop-in replacement for minimalBlock_groupTowerData_from_kernelRoute83Closers.
