    Provenance-semantic Section 7 minimal-block route #

    This sidecar targets the four Section 7 residuals that still appear in the current final closure:

    • minimalBlock_kernelModuleConcreteProvenanceData;
    • minimalBlock_rawHeadQuotientNoTrivialData;
    • minimalBlock_rawHeadSimplicityActualData;
    • minimalBlock_rawLowerScalarQuotientData.

    The replacement below is one same-tower semantic packet. It fixes the concrete subgroup/module provenance for R = Phi(K) <= K, a coherent lower tower derived from that provenance, and the actual head/lower scalar facts on that selected raw tower. The final route consumes only the resulting MinimalBlockGroupTowerData, so the four older residual names can be kept out of presentation_correct.
