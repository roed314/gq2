    Route34 minimal-block pushout/determinant provider #

    Route33 uses the route31 provider, whose pushout frontier is the four same-family central-pushout leaves

    • minimalBlock_preheadCentralPushoutQuadraticFamilyData;
    • minimalBlock_preheadCentralPushoutSquareDiagonalData;
    • minimalBlock_preheadCentralPushoutPolarSymmData;
    • minimalBlock_preheadCentralPushoutSimpleHeadDeterminantActualData.

    This sidecar gives a route31-compatible provider from one coherent manuscript packet: the actual scalar central-extension family K_lambda = K / ker lambda. That packet already contains the phi_lambda family, the actual square map, the diagonal and polar identities, naturality, and the simple-head determinant annihilation/radical conclusions. The route34 hook projects those fields back into route31's standard square/polar plus actual determinant interface, so the four route33 residual names disappear from the provider closure.

    For coordinators who prefer the finer Section 7 split, this file also exports a route34-named hook through the seven split actual central-extension packets from MinimalBlockActualCentralExtensionPushoutProofs.
