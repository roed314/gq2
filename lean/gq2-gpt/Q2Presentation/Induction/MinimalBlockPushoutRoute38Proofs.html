    Route38 minimal-block pushout/determinant provider #

    Route37 exposes seven Section 7 pushout residuals:

    • minimalBlock_preheadCentralPushoutQuadraticFamilyData;
    • minimalBlock_preheadCentralPushoutSquareDiagonalData;
    • minimalBlock_preheadCentralPushoutPolarSymmData;
    • minimalBlock_preheadCentralPushoutNaturalitySemanticData;
    • minimalBlock_preheadCentralPushoutDiagonalZeroSemanticData;
    • minimalBlock_preheadCentralPushoutPolarZeroSemanticData;
    • minimalBlock_preheadCentralPushoutRadicalContainmentSemanticData.

    The manuscript's determinant pushout is not seven independent choices. It is one actual scalar central-pushout family K_lambda = K / ker lambda, together with the simple-head determinant theorem for the same family.

    This Route38 sidecar records exactly that boundary:

    • actual scalar central-pushout data on the pre-head tower;
    • prop:simpleheaddet for the semantic family obtained from that actual scalar pushout.

    The old seven packets are projections from these two same-family packets. This does not claim the quotient groups K_lambda have been constructed in Lean; it strictly lowers the Route37 residuals to the manuscript-shaped actual scalar pushout and actual simple-head determinant packets already present in the API.
