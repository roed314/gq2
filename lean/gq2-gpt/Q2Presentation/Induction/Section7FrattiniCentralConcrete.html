    Section 7 Frattini centrality, exported to the concrete kernel packets #

    The corrected Section 7 trunk proves

    kernel_commutator_frattini_eq_bot : ⁅K.Ksub, kernelFrattini K⁆ = ⊥

    for the manuscript kernel K : KernelMinimal chief.toNonScalarChiefFactor attached to a first non-scalar chief factor. This sidecar pushes that theorem back through the older MinimalBlockKernelModuleConcreteFrattiniData interfaces:

    • first, a least-over-chief kernel packet is converted to the corrected KernelMinimal packet. This conversion is theorem-level: the old leastness field is stronger than the corrected minimal_inside field, so no new axiom is needed;
    • then the commutator equality is converted to the elementwise field frattini_central_in_K;
    • finally, the remaining Frattini-collapse packet is split so that the exponent-two field is obtained from the new centrality theorem plus the already-isolated fourth-power and square/commutator-layer inputs. Thus a direct closure of K_fourth_eq_one is left as the next independent manuscript lemma, rather than being entangled with centrality.

    This file is route-neutral: it introduces no axioms and does not edit existing routed files. It is intended to be imported by the next root route when replacing the old minimalBlock_kernelModuleConcreteFrattiniCentralData residual.
