    Candidate-side Arf / Gauss-sum: de-axiomatization of candidate_base_zeroCount #

    This file targets the axiom Q2Presentation.Quadratic.candidate_base_zeroCount (Quadratic/GaussSign.lean, l.418), the Lean rendering of manuscript prop:candidatezero: a nonsingular quadratic form over F₂ in dimension d has 2^{d-1} + (-1)^{Arf} 2^{d/2-1} zeros, with the sign determined by the Arf invariant of the form.

    STATUS (post-fix). The inconsistency diagnosed in this file has been corrected in Quadratic/GaussSign.lean: the two count axioms are now the consistent existential form ∃ ram, # = baseZeroCount ram d (the ram recovered as the form-tied gaussType), so the project no longer derives False. This file is retained as the diagnosis/record; the single now-stale False derivation (project_axiom_yields_false) is neutralized below.

    Outcome: the verbatim free-ram axiom statement was FALSE (now fixed) #

    The axiom is

    (qA : QuadF2 V) (hns : qA.Nonsingular) (ram : Bool) :
    (numZeros qA : ℤ) = baseZeroCount ram (Module.finrank (ZMod 2) V)

    Its bug: ram : Bool is a free argument, decoupled from qA. The manuscript's sign is (-1)^{Arf(Q)}, where the ramification flag is a property of the form (unramified ⟹ Arf=1 ⟹ minus type; ramified ⟹ Arf=0 ⟹ plus type). As written, the axiom asserts that the single integer numZeros qA equals baseZeroCount ram d for both ram = true and ram = false, forcing baseZeroCount true d = baseZeroCount false d — contradicting baseZeroCount_ne (proven in the same file). So no theorem candidate_base_zeroCount_proven with the verbatim statement can exist: it would prove False.

    This file establishes that rigorously, axiom-clean:

    • hyp — the hyperbolic plane q(a,b)=ab on (ZMod 2)², a genuinely nonsingular QuadF2 (hyp_nonsingular), with concrete zero count numZeros hyp = 3 (numZeros_hyp);
    • hyp_count_eq_plus / hyp_count_ne_minus — its true count is the plus type (3), not the minus type (1): the bug made concrete;
    • candidate_base_zeroCount_unsound — the verbatim statement (at V = (ZMod 2)²) implies False (closure ⊆ {propext, Classical.choice, Quot.sound}: the statement is outright refutable, not merely unprovable);
    • project_axiom_yields_false (NEUTRALIZED below — the bug is fixed) — this once derived False from the actual project axiom (the free ram instantiated at both values). With the corrected ∃ ram, … axiom it no longer typechecks; project_axiom_no_longer_yields_false records the now-sound existential in its place.

    The honest replacement #

    hyp_correct_existential records the correct shape (∃ ram, numZeros = baseZeroCount ram d, ram tied to the form) for the witness. The GENERAL corrected theorem — that this ∃ holds for every nonsingular QuadF2 — is the Dickson zero-count via the Arf invariant + Witt/Wall decomposition over F₂. mathlib has neither Arf nor the quadratic-form zero count, so the general statement must be built from scratch; it is out of scope for a verbatim de-axiomatization (which is impossible regardless). GaussSign.lean has now been fixed to constrain ram to the form — the existential candidate_base_zeroCount/q2_local_base_zeroCount plus the form-tied gaussType — so the downstream candidate_gauss_eq_local_gauss is now sound and the project is consistent again.

          The exact statement of candidate_base_zeroCount is FALSE.

          The axiom (GaussSign.lean l.418) takes the ramification flag ram : Bool as a free argument decoupled from the form qA, asserting # qA⁻¹(0) = baseZeroCount ram d for every ram. But numZeros qA is a single integer while baseZeroCount false d ≠ baseZeroCount true d (baseZeroCount_ne, same file). Instantiating at the genuinely nonsingular hyperbolic form hyp on (ZMod 2)² forces baseZeroCount false 2 = baseZeroCount true 2, i.e. 1 = 3 — a contradiction.

          Hence the axiom cannot be de-axiomatized as stated: there is no candidate_base_zeroCount_proven with the verbatim statement (it would prove False). The manuscript (prop:candidatezero, # Q⁻¹(0) = 2^{d-1} + (-1)^{Arf(Q)} 2^{d/2-1}) ties the sign to Arf(Q); the Lean axiom's bug is that ram is not constrained to the form's Arf invariant.
